## Object Detection in Images using YOLO11
```bash
!yolo task = detect mode = predict model = yolo11n.pt source = '/content/data/image.jpg'
1. task: detect, segment, classify
2. mode: train, predict, export, val
3. yolo11n: the model size (s < m < l < x)
   11n-x: x = (seg, pose, cls) correspond with task
4. source: = 
	- 'video.mp4'
	- '/content/my_folder/'
	- 'https://.../video.mp4'
	- 0 (调用本地摄像头，但在 Colab 云端无效，需特殊处理)
Tips: In "predict" mode, the default parameter
	- conf = 0.25(置信度阈值)
	    if the probability of this this object is target object, it will not
	    draw the frame
	- save = True 
```

### Show the video
```python
from IPython.display import HTML
from base64 import b64encode
import os
# Input video path
save_path = '/content/runs/detect/predict2/video.avi'
# Compressed video path
compressed_path = "/content/result_compressed.mp4"
os.system(f"ffmpeg -i {save_path} -vcodec libx264 {compressed_path}")

# Show video
mp4 = open(compressed_path,'rb').read()
data_url = "data:video/mp4;base64," + b64encode(mp4).decode()
HTML("""
<video width=400 controls>
	<source src="%s" type="video/mp4">
</video>
""" % data_url)

Tip:
1. Transcoding
   -ffmpeg: using ffmpeg(Fast Forward Moving Picture Experts Group) to recompile
   -i: input the video you save
   -vcodec libx264: recoding the video to H.264 format
2. Base64 Encoding
   -open('path', 'rb').read(): using binary mode to read the MP4 file
   -data: header
   -video/mp4: MIME Class, if it was image, thus, image/png
   -base64: Encoding method
   -,: add the data we want to decode
3. Type Conversion
   -b64encode: translating this "binary gibberish" into a safe string made of
    64 universal characters
   -.decode(): converts the bytes type into a standard String type	

4. HTML
   -HTML: use to generate a <video> tag in NoteBook output block
   -src="%s" type="video/mp4" %data_url: transit the str to player's source
```
